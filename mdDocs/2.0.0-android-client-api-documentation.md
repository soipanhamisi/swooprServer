## 1.0 System Overview

### 1.1 Platform Identity

**Swoopd** is a campus-focused carpooling platform exclusively for **USIU-Africa verified students**. The backend service enables students to:

- Authenticate securely with email verification
- Create and manage carpool trips
- Join existing carpools with compatible routes
- Receive real-time notifications via Firebase Cloud Messaging
- Track trip status and passenger information

### 1.2 Architecture Overview

```
Android Client App
        ↓
    HTTP/REST API (JSON)
        ↓
    Spring Boot Backend (Java 25)
        ↓
    JWT Authentication
        ↓
    MySQL Database
        ↓
    External APIs (Google Maps, Firebase)
```

### 1.3 Technology Stack

- **Backend:** Java 25.0.2, Spring Boot 3.x
- **Database:** MySQL 8
- **Authentication:** JWT (Bearer Token)
- **Push Notifications:** Firebase Cloud Messaging
- **External APIs:** Google Maps Routes & Geocoding API

### 1.4 Supported Platforms

- Android (minimum API level: TBD)
- Uses standard HTTP/REST API (can be consumed by any client)

## 2.0 Authentication & Security

### 2.1 Authentication Flow

The Swoopd platform uses **Email-based OTP (One-Time Password)** authentication with JWT tokens for session management.

#### Step 1: Request OTP

```
POST /auth/getOtp
Content-Type: application/json

{
  "email": "student@usiu.ac.ke"
}

Response: 200 OK
{
  "success": true,
  "message": "OTP sent to email",
  "data": null
}
```

- OTP is sent to the student's email address
- OTP is valid for a limited time (TBD - typically 10-15 minutes)
- User must have a valid USIU email address

#### Step 2: Verify OTP & Get Token

```
POST /auth/getNewToken
Content-Type: application/json

{
  "email": "student@usiu.ac.ke",
  "otp": 123456
}

Response: 200 OK
{
  "success": true,
  "message": "Authentication successful",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 604800,  // 7 days in seconds
    "tokenType": "Bearer"
  }
}
```

- Token is valid for **7 days**
- Must be included in all subsequent requests

#### Step 3: Refresh Token (Optional)

Before token expires, client can refresh it:

```
POST /auth/refreshToken
Content-Type: application/json

{
  "email": "student@usiu.ac.ke"
}

Response: 200 OK
{
  "success": true,
  "message": "Token refreshed",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 604800,
    "tokenType": "Bearer"
  }
}
```

### 2.2 Authenticated Requests

All protected endpoints require the JWT token in the `Authorization` header:

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json
```

**Example using Java/Kotlin (Retrofit):**

```kotlin
// Add interceptor to include token in all requests
val client = OkHttpClient.Builder()
    .addInterceptor { chain ->
        val originalRequest = chain.request()
        val newRequest = originalRequest.newBuilder()
            .header("Authorization", "Bearer $token")
            .build()
        chain.proceed(newRequest)
    }
    .build()

val retrofit = Retrofit.Builder()
    .baseUrl("http://localhost:8080")
    .client(client)
    .addConverterFactory(GsonConverterFactory.create())
    .build()
```

### 2.3 Security Requirements

✅ **Verified USIU Students Only**
- Must have valid USIU email (Outlook-based domain)
- Email verification required
- Active account status

✅ **SSL/TLS Encryption**
- All production requests must use HTTPS
- Self-signed certificates not allowed in production

✅ **Token Storage (Android)**
- Store JWT in Android Keystore or EncryptedSharedPreferences
- Never log or expose token
- Clear token on logout

✅ **Sensitive Data Protection**
- Passwords never exposed
- Tokens never logged
- Database schema never exposed
- Internal IDs minimized in responses

#### 3.2.3 Join Carpool

| Property | Value |
|----------|-------|
| **Method** | POST |
| **Endpoint** | `/trips/joinCarPool` |
| **Auth Required** | ✅ Yes |
| **Role Required** | `RIDE_SEEKER` |

**Request Body:**

```json
{
  "departureTime": "2026-07-11T14:00:00Z",
  "rsOriginDestination": {
    "originLatitude": -1.2836,
    "originLongitude": 36.8172,
    "destinationLatitude": -1.3163,
    "destinationLongitude": 36.7832
  }
}
```

**Field Descriptions:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `departureTime` | ISO 8601 Timestamp | ✅ | Preferred departure time (UTC) |
| `rsOriginDestination.*` | Coordinates | ✅ | Ride seeker's origin and destination |

**Response - Success (Trip Found) (200 OK):**

```json
{
  "success": true,
  "message": "Successfully joined carpool",
  "data": {
    "tripId": "770e8400-e29b-41d4-a716-446655440002",
    "hostName": "Jane Smith",
    "capacity": 2,
    "departureTime": "2026-07-11T14:00:00Z",
    "status": "OPEN",
    "destinationZone": "Kileleshwa",
    "passengerCount": 2,
    "routePolyline": "encoded_polyline_string_here"
  }
}
```

**Response - Backlog Entry (Trip Not Found) (202 Accepted):**

```json
{
  "success": true,
  "message": "Added to waitlist. You'll be notified when a compatible trip is available",
  "data": {
    "backlogEntryId": "880e8400-e29b-41d4-a716-446655440003",
    "destinationZone": "Kileleshwa",
    "requestMadeAt": "2026-07-11T10:30:45Z",
    "status": "WAITING_FOR_MATCH"
  }
}
```

**Matching Algorithm:**
1. System searches for OPEN trips in same destination zone
2. Departure time must be within configurable window (TBD)
3. If match found: User added to trip, notification sent
4. If no match: User added to backlog, waiting for compatible trip
5. When new trip created, backlog users automatically matched

**Error Cases:**
- `400 Bad Request` - Invalid coordinates or time
- `401 Unauthorized` - Not authenticated
- `403 Forbidden` - User is not a RIDE_SEEKER
- `409 Conflict` - User already in active carpool

**Implementation Example (Kotlin):**

```kotlin
data class JoinCarpoolRequest(
    val departureTime: String,
    val rsOriginDestination: Coordinates
)

interface SwoopApi {
    @POST("trips/joinCarPool")
    suspend fun joinCarpool(
        @Body request: JoinCarpoolRequest
    ): ApiResponse<CarpoolJoinResponse>
}

// Usage with error handling
try {
    val response = api.joinCarpool(JoinCarpoolRequest(
        departureTime = "2026-07-11T14:00:00Z",
        rsOriginDestination = userCoordinates
    ))
    
    when (response.statusCode) {
        200 -> {
            // Successfully joined trip
            val tripData = response.data
            showTripDetails(tripData)
        }
        202 -> {
            // Added to waitlist
            val backlogData = response.data
            showWaitlistNotification(backlogData)
        }
    }
} catch (e: Exception) {
    handleError(e)
}
```

## 5.0 Error Handling & Responses

### 5.1 HTTP Status Codes

| Status | Meaning | Common Causes |
|--------|---------|---------------|
| **200** | OK | Request successful |
| **201** | Created | Resource successfully created |
| **202** | Accepted | Request accepted but processing (e.g., added to backlog) |
| **400** | Bad Request | Invalid input, malformed JSON, missing fields |
| **401** | Unauthorized | Missing or invalid authentication token |
| **403** | Forbidden | Authenticated but not authorized (wrong role) |
| **404** | Not Found | Resource doesn't exist |
| **409** | Conflict | Resource already exists (duplicate email, vehicle, etc.) |
| **429** | Too Many Requests | Rate limit exceeded |
| **500** | Server Error | Internal server error |
| **503** | Service Unavailable | Server maintenance or temporary outage |

### 5.2 Error Response Format

### 5.3 Handling Errors in Android

**Kotlin Implementation:**

```kotlin
sealed class Result<out T> {
    data class Success<T>(val data: T) : Result<T>()
    data class Error(val exception: Exception) : Result<Nothing>()
    object Loading : Result<Nothing>()
}

// Usage in repository
suspend fun createTrip(tripRequest: CreateTripRequest): Result<TripData> {
    return try {
        val response = api.createTrip(tripRequest)
        if (response.success) {
            Result.Success(response.data)
        } else {
            Result.Error(ApiException(response.message))
        }
    } catch (e: Exception) {
        Result.Error(e)
    }
}

// Usage in ViewModel
viewModelScope.launch {
    when (val result = repository.createTrip(tripRequest)) {
        is Result.Success -> {
            updateUI(result.data)
        }
        is Result.Error -> {
            showError(result.exception.message)
        }
        is Result.Loading -> {
            showLoadingIndicator()
        }
    }
}
```

### 5.4 Retry Strategy

For transient errors (5xx, network timeouts):

```kotlin
suspend inline fun <T> withRetry(
    maxRetries: Int = 3,
    delayMs: Long = 1000,
    backoff: Boolean = true,
    crossinline block: suspend () -> T
): T {
    var currentDelay = delayMs
    var lastException: Exception? = null
    
    repeat(maxRetries) {
        return try {
            block()
        } catch (e: Exception) {
            lastException = e
            delay(currentDelay)
            if (backoff) currentDelay *= 2
        }
    }
    
    throw lastException ?: Exception("Unknown error")
}

// Usage
val tripData = withRetry {
    api.createTrip(tripRequest)
}
```

```kotlin
class SwoopMessagingService : FirebaseMessagingService() {
    
    override fun onMessageReceived(remoteMessage: RemoteMessage) {
        Log.d("FCM", "Message received: ${remoteMessage.data}")
        
        val title = remoteMessage.data["title"] ?: "Swoopd Notification"
        val body = remoteMessage.data["body"] ?: ""
        val tripId = remoteMessage.data["tripId"]
        
        showNotification(title, body, tripId)
    }
    
    override fun onNewToken(token: String) {
        Log.d("FCM", "Token refreshed: $token")
        // Send new token to backend
        updateTokenOnBackend(token)
    }
    
    private fun showNotification(title: String, body: String, tripId: String?) {
        val notification = NotificationCompat.Builder(this, CHANNEL_ID)
            .setSmallIcon(R.drawable.ic_notification)
            .setContentTitle(title)
            .setContentText(body)
            .setAutoCancel(true)
            .setContentIntent(
                PendingIntent.getActivity(
                    this, 0,
                    Intent(this, TripDetailsActivity::class.java).apply {
                        putExtra("tripId", tripId)
                    },
                    PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
                )
            )
            .build()
        
        NotificationManagerCompat.from(this)
            .notify(NOTIFICATION_ID, notification)
    }
    
    companion object {
        private const val CHANNEL_ID = "swoopd_trips"
        private const val NOTIFICATION_ID = 1
    }
}
```

## 8.0 Common Issues & Troubleshooting

### 8.1 Authentication Issues

**Problem: "OTP is incorrect or expired"**
- Verify OTP was entered correctly
- Check OTP expiration time (typically 10-15 minutes)
- Request new OTP if expired
- Check spam folder if email not received

**Problem: "Token is invalid" after login**
- Ensure token is stored correctly after authentication
- Check token doesn't have extra whitespace
- Verify token not expired (7-day validity)
- Refresh token before expiration

**Problem: 401 Unauthorized on protected endpoints**
- Verify token is included in Authorization header
- Check token format: `Bearer {token}`
- Ensure token hasn't expired
- Re-authenticate if token invalid

### 8.2 Trip Management Issues

**Problem: "Only carpool hosts can create trips"**
- User role must be `CARPOOL_HOST`
- Must have registered a vehicle first
- Check role in user profile

**Problem: "User already in active carpool"**
- User cannot join multiple trips simultaneously
- Cancel current trip or wait for completion
- Check trip status

**Problem: "No matching trip found" (added to backlog)**
- System searching for compatible trips in same zone
- Departure time may be outside matching window
- Will be notified when match found
- Can request new search with different time

**Problem: Invalid coordinates**
- Verify latitude/longitude format (decimal degrees)
- Latitude range: -90 to 90
- Longitude range: -180 to 180
- Use Google Maps or GPS to get precise coordinates

### 8.3 Firebase Integration Issues

**Problem: FCM token not refreshing**
- Ensure Firebase is properly initialized
- Check AndroidManifest.xml has correct permissions
- Verify google-services.json is in app directory
- Check build.gradle includes Google Services plugin

**Problem: Notifications not received**
- Verify FCM token sent to backend
- Check device has internet connectivity
- Verify app has notification permissions (Android 13+)
- Check notification channel is created
- Enable notifications in app settings

**Problem: "firebaseMessagingToken required"**
- Obtain FCM token before registration
- Ensure token format is correct (long string)
- Don't include "Bearer" or extra text

### 8.4 Network Issues

**Problem: Connection timeout**
- Check backend is running
- Verify backend URL is correct
- Check network connectivity
- Implement retry logic with exponential backoff

**Problem: SSL certificate error**
- Ensure using HTTPS in production
- Certificate must be valid and not self-signed
- Check system clock is correct

**Problem: JSON parsing error**
- Verify response format matches expected DTO
- Check for null values
- Ensure proper JSON encoding/decoding
- Log raw response for debugging

### 8.5 Data Validation Issues

**Problem: "Email is not valid USIU email"**
- Must use @usiu.ac.ke domain
- Verify email spelling
- Ensure email is registered with USIU

**Problem: "Invalid vehicle registration"**
- Kenya format: e.g., "KCF 123A"
- Check format matches pattern
- Verify registration number valid with NTSA

**Problem: "Capacity must be between 1 and 7"**
- Carpool capacity limited by vehicle type
- 4-seater: capacity 3 (1 driver + 3 passengers)
- 7-seater: capacity 6 (1 driver + 6 passengers)

### 8.6 Debugging Tips

**Enable verbose logging:**

```kotlin
// Retrofit logging interceptor
val loggingInterceptor = HttpLoggingInterceptor().apply {
    level = HttpLoggingInterceptor.Level.BODY
}

val client = OkHttpClient.Builder()
    .addInterceptor(loggingInterceptor)
    .build()
```

**Capture all requests/responses:**

```kotlin
// Custom interceptor
val debugInterceptor = Interceptor { chain ->
    val request = chain.request()
    Log.d("API", "Request: ${request.method} ${request.url}")
    Log.d("API", "Headers: ${request.headers}")
    
    val response = chain.proceed(request)
    Log.d("API", "Response: ${response.code}")
    Log.d("API", "Body: ${response.body?.string()}")
    
    response
}
```

- Set up environment variables for baseUrl and token
- Test each endpoint individually
- Verify response structure
